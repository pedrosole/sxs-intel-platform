# Story 2.7 — Fluxo Integrado Calendário → Design

**Epic:** 2 — Design Studio
**Tipo:** Feature
**Prioridade:** MEDIUM
**Estimativa:** P (pequena)
**Depende de:** 2.5
**Status:** Draft

---

## Descrição

Conectar o calendário editorial ao Design Studio. Quando uma peça é aprovada no calendário, ela aparece automaticamente na fila do Design Studio. Botões de ação conectam as duas telas.

## Acceptance Criteria

- [ ] Calendário (`/calendario/[token]`): botão "Criar Visual" em peças aprovadas
  - Redireciona para `/clientes/[slug]/design?piece={pieceId}`
  - Botão só aparece se `status = 'aprovado'`
- [ ] Página `/clientes`: coluna "Visual" com contadores
  - `3/12 peças com visual` ou badge de progresso
- [ ] Design Studio: deep link funciona
  - Se `?piece={id}` na URL → abre direto no painel dessa peça
- [ ] Status sincronizado:
  - Calendário mostra status visual da peça (ícones diferenciados)
  - `aprovado` = aguardando visual (azul)
  - `em_design` = em produção visual (amarelo)
  - `visual_aprovado` = visual pronto (verde)
  - `exportado` = PNG gerado (verde + ícone download)
- [ ] Navegação de volta: Design Studio → link para calendário do cliente

## Scope

**IN:** Botões de ação, deep links, status visual sincronizado, contadores
**OUT:** Notificações, automação de fluxo (auto-criar visual ao aprovar)

## Notas Técnicas

- Status expandido de `calendar_pieces` (migration na story 2.1)
- Usar `router.push` para navegação client-side
- Contadores via query agregada: `SELECT status, count(*) FROM calendar_pieces GROUP BY status`
