## Descrição

API route para geração de imagens temáticas via Gemini API (gemini-2.5-flash-image). Recebe prompt contextual + dados da marca, retorna imagem salva no Storage.

## Acceptance Criteria

- [x] `POST /api/design/generate-image` implementado
  - Input: `{ clientSlug, prompt, pieceId? }`
  - Carrega colors do cliente para enriquecer prompt
  - Chama Gemini API (`gemini-2.5-flash-preview-05-20`, `responseModalities: ['IMAGE']`)
  - Salva PNG no Supabase Storage (`generated-images/{slug}/`)
  - Retorna URL da imagem + metadata
- [x] Prompt builder: função que monta prompt contextual
  - Recebe: tema da peça, cores da marca, tom do brand voice
  - Adiciona: "No text, no watermarks, no logos. Vertical 3:4. Photorealistic."
  - Injeta cores como referência
- [x] Retry em caso de rate limit (429) — max 3 retries com backoff (15s/60s)
- [x] Erro tratado se API key inválida ou billing inativo
- [x] Timeout de 120s (maxDuration = 120)

## Scope

**IN:** API route, prompt builder, Storage save, error handling
**OUT:** UI de geração (story 2.5), regeneração interativa

## Notas Técnicas

- `GEMINI_API_KEY` já existe no `.env.local`
- Modelo: `gemini-2.5-flash-image` via `@google/generative-ai` (já instalado)
- Output: PNG ~1MB, ~1024x1024
- Runtime: Node.js (não Edge) — precisa de buffer manipulation
- `route.ts` com `export const maxDuration = 120`
