# Story 2.5 — Design Studio Page

**Epic:** 2 — Design Studio
**Tipo:** Feature
**Prioridade:** HIGH
**Estimativa:** G (grande)
**Depende de:** 2.2, 2.3, 2.4
**Status:** Draft

---

## Descrição

Página principal do Design Studio onde o usuário visualiza peças aprovadas, gera imagens de fundo, monta o visual da peça, faz preview e aprova. É o coração da integração @leo na plataforma.

## Acceptance Criteria

### Página
- [ ] Rota: `/clientes/[slug]/design`
- [ ] Link no calendário: peça aprovada → botão "Criar Visual" → redireciona aqui
- [ ] Link na página `/clientes`: ícone de design ao lado de cada cliente

### Lista de Peças
- [ ] Listar peças com `status = 'aprovado'` do calendário (aguardando visual)
- [ ] Cada peça mostra: título, formato (post/carrossel), data, status visual
- [ ] Status visual: pendente | gerando | preview | aprovado | exportado
- [ ] Clicar em uma peça → abre painel de criação

### Painel de Criação (peça selecionada)
- [ ] Exibe conteúdo textual da peça (título, body, CTA, legenda)
- [ ] Seção "Imagem de Fundo":
  - Textarea para prompt (pré-preenchido com sugestão baseada no tema)
  - Botão "Gerar Imagem" → chama API 2.4
  - Preview da imagem gerada
  - Botão "Regenerar" para tentar novamente
- [ ] Seção "Logo":
  - Grid com variantes disponíveis (do asset management)
  - Seleção automática por contraste (escuro→white, claro→color)
  - Override manual possível
- [ ] Seção "Fonte":
  - Mostra fonte configurada (própria ou Google Fonts)
  - Badge: "Fonte própria" vs "Aproximação"
- [ ] Botão "Gerar Preview" → monta HTML completo:
  - Imagem de fundo como base64 (do Storage)
  - Logo selecionada como base64
  - Fontes como @font-face base64 (se própria) ou Google Fonts link
  - Conteúdo textual mapeado
  - Overlay escuro automático
- [ ] Preview inline: iframe 420×525px com o HTML gerado
- [ ] Botão "Aprovar Visual" → status → `visual_aprovado`
- [ ] Campo de feedback para ajustes

### Para Carrossel (multi-slide)
- [ ] Navegação entre slides (prev/next ou thumbnails)
- [ ] Imagem de fundo pode ser por slide ou compartilhada
- [ ] Progress bar e swipe arrow renderizados

### Registro
- [ ] Criar/atualizar registro em `design_pieces` a cada ação
- [ ] Salvar HTML completo no banco
- [ ] Salvar prompt usado, logo variant, revision count

## Scope

**IN:** Página, lista de peças, geração de imagem, montagem HTML, preview iframe, aprovação
**OUT:** Export PNG (story 2.6), edição avançada de layout

## Notas Técnicas

- HTML da peça gerado server-side via API route (`/api/design/generate-piece`)
- iframe sandbox para preview seguro
- Base64 de imagem + logo + fonte pode resultar em HTML de 2-5MB — normal
- Usar `URL.createObjectURL(new Blob([html]))` para iframe src se necessário
- Glass cards + dark theme consistente com o resto da plataforma
