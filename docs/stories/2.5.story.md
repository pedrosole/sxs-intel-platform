## Descrição

Página principal do Design Studio onde o usuário visualiza peças aprovadas, gera imagens de fundo, monta o visual da peça, faz preview e aprova. É o coração da integração @leo na plataforma.

## Acceptance Criteria

### Página
- [x] Rota: `/clientes/[slug]/design`
- [x] Link na página `/clientes/[slug]`: botão "Design Studio"
- [x] API `/api/calendario/pieces?slug=` para listar peças aprovadas

### Lista de Peças
- [x] Listar peças com status aprovado/em_design/visual_aprovado do calendário
- [x] Cada peça mostra: título, formato, data, status visual com cores
- [x] Clicar em uma peça → abre painel de criação

### Painel de Criação (peça selecionada)
- [x] Exibe conteúdo textual da peça (título, cluster, objetivo, caption, CTA)
- [x] Seção "Imagem de Fundo": textarea com prompt auto-sugerido, gerar/regenerar
- [x] Seção "Logo": seleção de variante (auto/white/cropped/original)
- [x] Seção "Fonte": badge "Fonte propria" vs "Google Fonts"
- [x] Botão "Gerar Preview" → monta HTML via `/api/design/generate-piece`
  - Imagem fundo base64, logo base64, fontes @font-face, overlay escuro
- [x] Preview inline: iframe 420×525px com HTML gerado
- [x] Botão "Aprovar Visual" → status visual_aprovado
- [x] Campo de feedback para ajustes

### Para Carrossel (multi-slide)
- [ ] Navegação entre slides — adiado para iteração futura

### Registro
- [x] Criar/atualizar registro em `design_pieces` a cada ação
- [x] Salvar HTML completo no banco
- [x] Salvar prompt usado, logo variant, revision count

## Scope

**IN:** Página, lista de peças, geração de imagem, montagem HTML, preview iframe, aprovação
**OUT:** Export PNG (story 2.6), edição avançada de layout

## Notas Técnicas

- HTML da peça gerado server-side via API route (`/api/design/generate-piece`)
- iframe sandbox para preview seguro
- Base64 de imagem + logo + fonte pode resultar em HTML de 2-5MB — normal
- Usar `URL.createObjectURL(new Blob([html]))` para iframe src se necessário
- Glass cards + dark theme consistente com o resto da plataforma
