# Story 1.3 — Rewrite /clientes Page with Supabase Integration

**Status:** Done
**Tipo:** Feature
**Prioridade:** HIGH
**Estimativa:** M (media)

---

## Descricao

A pagina `/clientes` exibe dados mock hardcoded (`src/data/mock.ts`) com apenas 3 clientes estaticos (Marco, Triade, Marisa). Clientes registrados via Hermes (ex: Gabriela Lopes) existem no Supabase mas nao aparecem na UI.

A pagina precisa ser reescrita para:
1. Puxar clientes reais do Supabase
2. Mostrar historico de entregas (jobs) por cliente
3. Exibir links dos calendarios gerados
4. Mostrar briefings vinculados

---

## Escopo

### IN
- Criar API route `GET /api/clientes` que retorna clientes do Supabase com:
  - Dados do cliente (name, niche, status, instagram_handle, created_at)
  - Contagem de jobs (total, completados)
  - Ultimo job (data, status)
  - Links de calendarios ativos (share_token de calendar_meta)
- Reescrever `src/app/clientes/page.tsx` como Client Component que:
  - Faz fetch de `/api/clientes` no mount
  - Exibe cards por cliente com: nome, nicho, handle, status, data de cadastro
  - Mostra badge com total de entregas
  - Lista links dos calendarios com data (clicaveis → `/calendario/{token}`)
  - Mostra ultimo briefing (tipo + data)
  - Loading state e empty state
- Criar pagina de detalhe `src/app/clientes/[slug]/page.tsx` com:
  - Dados completos do cliente
  - Timeline de jobs (data, agentes, status)
  - Links de todos os calendarios
  - Briefings historicos

### OUT
- CRUD de clientes (criar/editar/deletar) — clientes sao criados via chat com Hermes
- Remover `src/data/mock.ts` (pode ser usado em outras paginas)

---

## Criterios de Aceitacao

- [x] AC1: `GET /api/clientes` retorna lista de clientes do Supabase com jobs e calendar links
- [x] AC2: Pagina `/clientes` exibe TODOS os clientes do banco (incluindo Gabriela Lopes)
- [x] AC3: Cada card mostra: nome, nicho, status, total de entregas, ultimo job
- [x] AC4: Links de calendario sao clicaveis e abrem `/calendario/{token}`
