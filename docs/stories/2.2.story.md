# Story 2.2 — Asset Upload API

**Epic:** 2 — Design Studio
**Tipo:** Feature
**Prioridade:** CRITICAL
**Estimativa:** M (média)
**Depende de:** 2.1
**Status:** Done

---

## Descrição

Criar API routes para upload, listagem e remoção de assets do cliente. Inclui processamento server-side de logos (crop + inversão).

## Acceptance Criteria

- [x] `POST /api/clientes/[slug]/assets` — upload multipart de arquivo
  - Aceita: PNG, JPG, SVG (logos), JSON (cores), TXT (fontes), TTF/OTF/WOFF2 (fontes), PNG/JPG (referências)
  - Salva no Supabase Storage (`client-assets/{slug}/{category}/`)
  - Registra em `client_assets`
  - Retorna asset criado com URL
- [x] `GET /api/clientes/[slug]/assets` — listar assets por categoria
  - Query param: `?category=logo|color|font|reference`
  - Retorna assets com URLs públicas do Storage
- [x] `DELETE /api/clientes/[slug]/assets/[id]` — remover asset
  - Remove do Storage + registro no banco
- [x] `POST /api/clientes/[slug]/assets/prepare` — processar logos
  - Crop whitespace (sharp trim)
  - Gerar versão branca (sharp negate)
  - Salvar variantes no Storage
  - Atualizar metadata no registro
  - Gerar manifest.json
- [x] Validação: tamanho máximo 10MB por arquivo
- [x] Validação: MIME type verificado (não confiar na extensão)

## Scope

**IN:** API routes, Storage upload, processamento sharp, validações
**OUT:** UI de upload (story 2.3)

## Notas Técnicas

- `formData.get('file')` para multipart no App Router
- Sharp roda server-side no Node.js runtime (não Edge)
- Crop: `sharp(buffer).trim().extend({...}).toBuffer()`
- Negate: `sharp(buffer).negate({alpha:false}).toBuffer()`
- Fontes: apenas armazenar — conversão base64 acontece na geração (story 2.5)
