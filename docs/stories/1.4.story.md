# Story 1.4 — Fix Hermes Redundancy (Stop Reorganizing Agent Outputs)

**Status:** Done
**Tipo:** Bug Fix
**Prioridade:** HIGH
**Estimativa:** P (pequena)

---

## Descricao

Atualmente, Hermes faz DUAS coisas desnecessarias apos os agentes entregarem:
1. **Reorganiza/resume** o que cada agente ja entregou formatado no chat — desperdicando tokens
2. **Nao entrega o link do calendario** — a unica acao util que deveria fazer no final

O briefing da Gabriela Lopes consumiu tokens excessivos porque Hermes repetiu informacoes que os agentes ja tinham entregado diretamente no stream.

**Comportamento esperado:**
- Hermes le o briefing → salva dados → delega aos agentes → cada agente entrega no chat diretamente
- No final do pipeline, Hermes APENAS entrega o link do calendario (se houver) e um resumo de 1-2 linhas ("Entrega concluida. X pecas no calendario: [link]")
- Para entregas sem calendario (analise, pesquisa, blog), Hermes nao precisa resumir — os agentes ja entregaram

---

## Escopo

### IN
- Atualizar prompt do Hermes em `system-prompts.ts` adicionando regra:
  - "Apos a diretiva, NAO resuma nem reorganize o que os agentes entregaram. Cada agente entrega diretamente no chat. Sua unica acao pos-pipeline e entregar o link do calendario (se aplicavel) e uma frase de conclusao."
- No `route.ts` (API chat), apos o pipeline/agents completarem:
  - NAO chamar Hermes novamente para resumir
  - Se houver evento `calendar_link`, emitir mensagem final com o link
  - Se nao houver, emitir mensagem simples: "Entrega concluida para {clientName}."

### OUT
- Mudancas no roteamento (Rota A/B/C)
- Mudancas nos prompts dos outros agentes
- Logica de re-execucao de pecas vetadas
