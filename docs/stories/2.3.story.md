## Descrição

Criar interface de gestão de assets na ficha do cliente com 4 abas: Logos, Cores, Fontes e Referências. Acessível via `/clientes/[slug]/assets`.

## Acceptance Criteria

### Navegação
- [x] Página `/clientes/[slug]/assets` criada
- [x] Link na página `/clientes/[slug]` para acessar assets ("Gerenciar Assets")
- [x] Tabs component com 4 abas: Logos | Cores | Fontes | Referências

### Tab 1 — Logos
- [x] Drag & drop ou botão para upload de logo (PNG/SVG)
- [x] Preview das logos uploadadas (thumbnail grid)
- [x] Botão "Preparar Assets" — chama API prepare, mostra progresso
- [x] Após preparo: mostra badges cropped/white
- [x] Delete individual

### Tab 2 — Cores
- [x] Color picker para cor primária
- [x] Botão "+" para adicionar cores secundárias com label (cream, gold, etc.)
- [x] Preview da paleta completa (swatches)
- [x] Auto-derivação dos 6 tokens (BRAND_PRIMARY → LIGHT_BG etc.)
- [x] Preview dos tokens derivados
- [x] Salvar como `colors.json` via API

### Tab 3 — Fontes
- [x] Upload de arquivos .ttf / .otf / .woff2
- [x] Preview tipográfico ao vivo com @font-face da fonte uploadada
- [x] Badge visual: "Fonte propria" vs "Google Fonts (aproximacao)"
- [x] Delete individual

### Tab 4 — Referências
- [x] Upload múltiplo de imagens (drag & drop batch)
- [x] Grid visual (4 colunas responsivo)
- [x] Lightbox para ver em tamanho maior
- [x] Delete individual

### Geral
- [x] Loading states em todos os uploads
- [x] Toast de confirmação/erro
- [x] Consistente com design system existente (glass cards, dark theme)

## Scope

**IN:** Página, 4 abas, upload, preview, delete, prepare-assets
**OUT:** Integração com Design Studio (story 2.5), edição avançada de imagens

## Notas Técnicas

- Usar `shadcn/ui` Tabs component
- Preview de fonte via `@font-face` injetado em `<style>` com URL do Storage
- Color picker: pode usar input[type=color] + hex text input
- Drag & drop: usar `onDrop` nativo + `e.dataTransfer.files`
